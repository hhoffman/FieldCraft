# Image Plan

## Principles
- Images must clarify, not decorate
- Prefer diagrams over photos where possible
- Each image should support a specific technique or decision

## Per Chapter
- 1–2 diagrams (required)
- 1–3 real-world examples (optional)

## Categories
- Shelter diagrams
- Fire layouts
- Navigation diagrams
- Signaling patterns
- Knot diagrams

## Storage
docs/assets/images/

## Naming Convention
<chapter>-<topic>-<type>.png

Example:
shelter-lean-to-diagram.png
fire-log-cabin-layout.png
